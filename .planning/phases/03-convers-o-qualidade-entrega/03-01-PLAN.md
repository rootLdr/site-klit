| navegador -> Formspree | Dados do formulario (nome/email/empresa/telefone/mensagem) cruzam para um servico externo de terceiro via HTTPS POST |
| bots -> formulario | Spam automatizado tenta submeter o form |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing/DoS (spam) | ContatoForm submit | mitigate | Honeypot `_gotcha` oculto; Formspree descarta envios preenchidos. Sem CAPTCHA neste ciclo (form de baixo volume) |
| T-03-02 | Information Disclosure | dados do form em transito | accept | Envio via HTTPS para Formspree (TLS); sem PII sensivel alem de contato comercial voluntario; sem armazenamento proprio |
| T-03-03 | Tampering | endpoint placeholder SEU_FORM_ID | accept | Endpoint e placeholder ate o usuario colar o formspreeId real; documentado no README (03-03). Enquanto placeholder, envio falha de forma controlada (estado de erro tratado) |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | Nenhum pacote novo instalado neste plano (Formspree e HTTP puro, sem SDK); sem superficie de supply-chain adicional |
